Getting Started

⚠️ First you must go to Flow Developer Portal for the general requirements and the project overview.

Cloning the Repo

The default branch of this project is the dev branch. In a case you want to open a PR, it should be opened to merge into the dev branch.

your@pc:~$ git clone https://github.com/FlowFintech/FlowFront -b dev

your@pc:~$ cd FlowFront

Install all dependencies

yarn install

While the Backend server is alrealdy running, start the server for the frontend.

yarn start

The Backend is built on Ruby on Rails, you need to have the server running, refer to the BackEnd Documentation to set the backend environment.


Contributing

Depending on whether your implementing a feature, doing a chore, fixing bugs or a hotfix, you must use the following branches naming convention: {type/short-description} Example

git checkout -b bug/fixing-layout
git checkout -b feature/adding-loan-details
git checkout -b chore/correct-typo
git checkout -b hotfix/refactor-jbuilder

Your feature/bug branch should be requested to merge into the dev branch and upon approval and merging of the PR, the branch should be deleted.


⚠️ Important: Do not install any dependency, if you feel you miss some library, create a discussion thread in #frontend channel in Slack

Commenting

Comment every Class/Function and other parts of the code you feel needs to be commented, this project is very dynamic and lots of devs come and go.

Flow uses TSDoc library to handle the comments and TypeDoc to generate the Documentation.

Use the custom tag @author in every class, function, file you create.
